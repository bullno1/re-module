Detailed Description

A single file library for live reloading.

In exactly one source file of the host program, define REMODULE_HOST_IMPLEMENTATION before including remodule.h:

#define REMODULE_HOST_IMPLEMENTATION
#include "remodule.h"
remodule.h
A single file library for live reloading.

Likewise, in exactly one source file of every plugin, define REMODULE_PLUGIN_IMPLEMENTATION before including remodule.h:

#define REMODULE_PLUGIN_IMPLEMENTATION
#include "remodule.h"

A plugin must define an entrypoint .

If a plugin has any global state that needs to be preserved across reloads, mark those with REMODULE_VAR.

#define REMODULE_VAR(TYPE, NAME)
Mark a global variable in the plugin for state transfer.
Definition: remodule.h:68
Parameters
TYPEThe type of the variable.
NAMEThe name of the variable. This must be unique within each plugin.
Remarks
If the type of the variable changes between reloads, it will not be preserved. The new instance will have the variable at its initial value.
Only a shallow copy will be made using memcpy to migrate data from the old plugin instance to the new one.
As long as the plugin uses the host's allocator or its allocator's state is preserved, everything should work out of the box.
On the other hand, pointers to static data in the plugin or structures allocated using the plugin's private allocator are problematic.
For more complex cases, make use of REMODULE_OP_BEFORE_RELOAD and REMODULE_OP_AFTER_RELOAD to serialize and deserialize. The target for serialization could be the userdata pointer in remodule_entry.

Load a module.

This will trigger REMODULE_OP_LOAD in the module's entrypoint .

Parameters
pathPath to the module.
userdataArbitrary userdata that will be passed to the entrypoint of the module.
Returns
The reloadable module.
Remarks
On Windows, due to file locking, instead of loading the module directly, a temporary copy will be made. This will be loaded instead of the original module. Therefore, the directory containing the module must be writable.
The temporary file will be deleted once it's no longer needed.
